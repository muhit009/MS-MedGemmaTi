MedGemma Clinical Suite - Setup Guide
======================================

Prerequisites:
- Python 3.10+
- Node.js 18+
- A Supabase project with the schema applied (see medgemma-backend/supabase/migrations/)


1. BACKEND SETUP
----------------

cd medgemma-backend
python -m venv venv

# Activate virtual environment:
source venv/Scripts/activate      # Windows (Git Bash)
source venv/bin/activate          # Mac/Linux

pip install -r requirements.txt

# Create your env file:
cp .env.example .env

# Fill in the following values in .env:
#   SUPABASE_URL          - Your Supabase project URL
#   SUPABASE_SERVICE_KEY  - Supabase service role key
#   JWT_SECRET            - Secret key for signing auth tokens
#   RUNPOD_API_KEY        - RunPod API key (optional, falls back to mock AI)
#   RUNPOD_ENDPOINT_ID    - RunPod endpoint ID (optional, falls back to mock AI)

# Start the backend:
uvicorn app.main:app --reload --port 8000


2. FRONTEND SETUP
-----------------

cd frontend
npm install

# Create your env file:
cp .env.example .env.local

# Fill in the following values in .env.local:
#   NEXT_PUBLIC_API_URL           - Backend URL (e.g. http://localhost:8000)
#   NEXT_PUBLIC_SUPABASE_URL      - Your Supabase project URL
#   NEXT_PUBLIC_SUPABASE_ANON_KEY - Supabase anon/public key

# Start the frontend:
npm run dev


3. ACCESS
---------

Frontend: http://localhost:3000
Backend:  http://localhost:8000
API Docs: http://localhost:8000/docs

Default login: dr.smith / password


4. NOTES
--------

- If RunPod credentials are not set, the AI will use mock responses for development.
- The Supabase project needs the storage bucket "medical-images" created.
- Apply the database migration in medgemma-backend/supabase/migrations/ to set up tables and seed data.
